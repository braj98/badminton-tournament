What Still Needs Improvement

Right now I see some duplication.

For example:

// models/tournament.js
createTournament()

// engine/state.js
defaultState()

These are both defining tournament structure in different places.

I would eventually have:

models/
    tournament.js
    participant.js
    match.js

be the ONLY place that defines data structures.

Then:

engine/

only manipulates them.

Biggest Architectural Issue Remaining

You still have:

let state = null;
let currentCategory = null;
let currentView = null;
let _isAdmin = false;

as globals.

For V1 this is okay.

For multi-sport V2 I'd create:

AppState = {
    tournament,
    currentCategory,
    currentView,
    user
}

and pass it around.

This will eliminate a lot of strange bugs.
