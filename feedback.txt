Tournament Manager UX Improvement - Match Status & Results Redesign
Objective

Simplify tournament navigation by separating:

Current Matches
Upcoming Matches
Completed Matches
Champions

Instead of combining everything under a single "Results" page.

This approach aligns with how spectators, players, and organizers naturally think during a tournament.

Problem Statement

Current Results page attempts to show:

Live matches
Upcoming matches
Completed matches
Champions
Tournament summary

This mixes:

Past
Present
Future

into a single view.

Result:

Harder navigation
User confusion
More complex UI logic
Difficult to scale as tournaments grow
User Mental Model

Most users ask one of four questions:

Question 1

What is happening right now?

Answer:

LIVE MATCHES

Question 2

What match is next?

Answer:

UPCOMING MATCHES

Question 3

Who won?

Answer:

CHAMPIONS

Question 4

What were the previous match results?

Answer:

RESULTS ARCHIVE

Proposed Navigation

Replace current Results-centric approach with:

🏆 Champions
🔥 Live
📅 Upcoming
📖 Results

Purpose:

Avoids guessing based on scores.

Example:

Bad Approach:

→ Assume match completed

Problems:

Corrections
Walkovers
Interrupted matches

Admin explicitly confirms completion.
