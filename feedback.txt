# Tournament Manager VNext - Event, Competition Template & Tournament Model

## Objective

Separate:

1. Reusable Definitions
2. Event Configuration
3. Tournament Data

This will make the platform easier to manage as more sports and tournaments are added.

---

# Current Model

Current structure:

Event
→ Category
→ Tournament

Example:

BREN AVALON SPORTS MEET 2026

* Junior
* Junior Doubles
* Senior Boys
* Senior Girls
* Senior Doubles

Problem:

Categories are tied to a specific event.

If a new event is created:

Summer Games 2027

the same categories need to be recreated.

This creates duplication.

---

# Proposed Model

Introduce three layers:

Competition Template
→ Event
→ Tournament Instance

---

# Layer 1 - Competition Templates

Competition Templates are reusable definitions.

Examples:

Badminton

* Junior Singles
* Junior Doubles
* Senior Boys
* Senior Girls
* Senior Doubles
* Adults Singles
* Adults Doubles

Table Tennis

* Open Singles
* Open Doubles

Chess

* Open Category

These templates are created once and reused forever.

---

## Competition Template Example

```json
{
  "id": "junior_singles",
  "name": "Junior Singles",
  "sport": "badminton",
  "type": "singles"
}
```

Another example:

```json
{
  "id": "tt_open",
  "name": "Open Singles",
  "sport": "tableTennis",
  "type": "singles"
}
```

---

# Layer 2 - Events

Events represent actual tournaments being organized.

Examples:

* BREN AVALON SPORTS MEET 2026
* Summer Games 2027
* Winter Games 2027

Events do not define categories.

Events only reference templates.

---

## Event Example

```json
{
  "id": "sports_meet_2026",
  "name": "BREN AVALON SPORTS MEET 2026",

  "competitionIds": [
    "junior_singles",
    "junior_doubles",
    "senior_boys",
    "senior_girls"
  ]
}
```

---

# Layer 3 - Tournament Instance

Tournament Instance contains actual runtime data.

Examples:

* Players
* Fixtures
* Standings
* Knockout
* Champion

This is what changes during tournament execution.

---

## Tournament Instance Example

```json
{
  "eventId": "sports_meet_2026",
  "competitionId": "junior_singles",

  "state": {
    "players": [],
    "groups": [],
    "fixtures": [],
    "knockout": {}
  }
}
```

---

# Benefits

## Reusability

Create category once.

Reuse across multiple events.

Example:

Junior Singles

can be used in:

* Sports Meet 2026
* Sports Meet 2027
* Summer Games 2027

without recreation.

---

## Easier Maintenance

Rules exist in one place.

If format changes:

Example:

Junior Singles

changes from:

Top 2 qualify

to:

Top 4 qualify

Only template changes.

---

## Multi-Sport Ready

Supports:

* Badminton
* Table Tennis
* Chess
* Carrom

without architecture changes.

# Recommendation

Do not make Event a property of Category.

Current:

Category
{
event
}

Future:

Event
{
competitionIds
}

This keeps categories reusable and prevents duplication.

The platform should treat Events as organizers of competitions, while Competition Templates define the reusable rules and structure.
