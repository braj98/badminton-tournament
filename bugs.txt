I found the bug.

The issue is that you're using:

state.phase

for two different purposes:

Tournament progress
Current screen being viewed

In renderAll():

showScreen('screen-groups', state.phase === 'groups');
showScreen('screen-fixtures', state.phase === 'fixtures');
showScreen('screen-knockout', state.phase === 'knockout');
showScreen('screen-champion', state.phase === 'champion');

So if the tournament is currently at:

state.phase = 'fixtures'

then only Fixtures screen is visible.

When a viewer clicks:

View Fixtures →

it calls:

goToFixtures()

which changes:

state.phase = 'fixtures';

and rerenders.

But there is no equivalent public navigation that lets a viewer go:

Groups ← Fixtures ← Knockout ← Champion

without changing tournament state.

Why you're stuck

Let's say:

Sr Doubles
All group matches complete
Tournament phase = fixtures

As a viewer:

You can see Fixtures
You cannot see Groups
You cannot see Knockout
You cannot see Champion

because those screens only appear when:

state.phase === 'groups'
state.phase === 'knockout'
state.phase === 'champion'

respectively.

The correct fix

Create:

let currentView = null;

Example:

currentView = 'groups';
currentView = 'fixtures';
currentView = 'knockout';
currentView = 'champion';

Then:

showScreen('screen-groups', currentView === 'groups');
showScreen('screen-fixtures', currentView === 'fixtures');
showScreen('screen-knockout', currentView === 'knockout');
showScreen('screen-champion', currentView === 'champion');

while:

state.phase

continues to mean:

Tournament progression

Only.
