Your HTML correctly marks the button as:

<button class="btn btn-secondary admin-only"
        onclick="toggleManagePanel()">
  ⚙️ Manage
</button>

and your CSS says:

body.viewer-mode .admin-only {
    display: none !important;
}

So in theory the Manage button should disappear.

The problem is likely here:

function showResultsPage() {
  ...
  renderResults();
  applyViewerMode();
  document.querySelectorAll('.admin-only')
          .forEach(el => el.style.display = 'none');
}

but inside renderResults() the first line is:

clearDisabled();

and clearDisabled() does:

document.body.classList.remove('viewer-mode');

...
app.querySelectorAll('.admin-only')
   .forEach(el => el.classList.remove('hidden'));

So your code is constantly fighting itself:

showResultsPage()
    ↓
applyViewerMode()

renderResults()
    ↓
clearDisabled()
    ↓
remove viewer-mode
    ↓
show admin controls again
Quick Test

Open browser DevTools and run:

document.body.classList.contains('viewer-mode')

If it returns:

false

while viewing Results page as a viewer, that's the bug.

Fix #1 (Recommended)

Modify clearDisabled():

Current:

document.body.classList.remove('viewer-mode');

Change to:

if (_isAdmin) {
    document.body.classList.remove('viewer-mode');
}

This prevents viewer mode from being removed accidentally.

Fix #2 (Even Better)

At the end of renderResults() add:

if (!_isAdmin) {
    applyViewerMode();
}

That guarantees admin controls stay hidden on Results screen.

Another bug I noticed

You currently have:

toggleManagePanel() {
   if (!_isAdmin) return;
